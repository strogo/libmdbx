546  * argument.
547  *
548  * Otherwise, if MDBX was builded not as a DLL, some black magic
549  * may be required depending of Windows version:
550  * - Modern Windows versions, including Windows Vista and later, provides
551  * support for "TLS Directory" (e.g .CRT$XL[A-Z] sections in executable

553  * and you do not need to call mdbx_dll_handler().
554  * - Obsolete versions of Windows, prior to Windows Vista, REQUIRES calling

557  * - This behavior is under control of the MODX_CONFIG_MANUAL_TLS_CALLBACK

559  * of Windows at build time.
560  * But you may override MODX_CONFIG_MANUAL_TLS_CALLBACK in special cases.
561  *
562  * Therefore, building MDBX as a DLL is recommended for all version of Windows.
563  * So, if you doubt, just build MDBX as the separate DLL and don't worry. */
